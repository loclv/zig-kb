# Migrating to Zig 0.16

Zig 0.16 (April 2026) introduces I/O as an Interface, a major language paradigm shift, along with substantial type system refinements and standard library cleanups.

## Table of Contents
- [Standard Library](#standard-library)
- [Language Changes](#language-changes)
- [Build System](#build-system)

## Standard Library

### I/O as an Interface

The biggest change in 0.16: I/O is now an interface-based abstraction, similar to how allocators work. You pass an I/O interface into functions rather than relying on concrete writer/reader types.
Old API (0.15):
```zig
fn printGreeting(writer: anytype) !void {
    try writer.print("Hello\n", .{});
}
```

New API (0.16+):
```zig
fn printGreeting(io: std.Io) !void {
    var writer = io.writer();
    try writer.print("Hello\n", .{});
}
```

Key types:
- `std.Io` — the interface handle
- `std.Io.Writer`, `std.Io.Reader` — concrete operations
- `std.Io.Future` — for async-style I/O scheduling
- `std.Io.Group` — for concurrent I/O operations

### "Juicy Main" — Simplified Entry Point

`main` can now receive an environment handle, eliminating boilerplate for allocators and argv.
Old API:
```zig
pub fn main() !void {
    var gpa = std.heap.GeneralPurposeAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();
    const args = try std.process.argsAlloc(allocator);
    defer std.process.argsFree(allocator, args);
    // ...
}
```

New API (0.16+):
```zig
pub fn main(env: std.process.Env) !void {
    const args = env.args;
    const allocator = env.allocator;
    // ...
}
```

Note: The classic `main()` signature still works; this is opt-in.

### ArenaAllocator Becomes Thread-Safe and Lock-Free

`std.heap.ArenaAllocator` is now lock-free and thread-safe by default. Remove any manual mutex wrapping around arena usage.

### ThreadSafe Allocator Removed

`std.heap.ThreadSafeAllocator` was deleted. Use `std.heap.ArenaAllocator` or manage synchronization manually.

### Environment Variables and Process Arguments

Environment variables and process arguments are no longer global singletons. Access them through `std.process.Env`.
Old API:
```zig
const home = std.process.getEnvVarOwned(allocator, "HOME");
```

New API:
```zig
const home = try env.get("HOME");
```

### Deflate Compression Added, Decompression Simplified

`std.compress.deflate` gains compression support. The decompression API was simplified.

### Debug Information Reworked

Debug info generation was rewritten. If you rely on custom DWARF output or debug plugins, verify compatibility.

## Language Changes

### `@Type` Replaced with Individual Type-Creating Builtins

The monolithic `@Type` builtin was split into specific builtins for each type kind.
Old API:
```zig
const MyInt = @Type(.{ .Int = .{ .signedness = .unsigned, .bits = 10 } });
```

New API:
```zig
const MyInt = @Int(.unsigned, 10);
```

Available builtins:
- `@Int(signedness, bits)`
- `@Float(bits)`
- `@Array(len, child)`
- `@Vector(len, child)`
- `@Pointer(...)` — see docs for full signature
- `@Struct(fields)`, `@Union(fields)`, `@Enum(fields)`

### `@cImport` Moving to Build System

`@cImport` is deprecated in source code. C header imports should be done in `build.zig` via `b.addTranslateC()`.
Old API (in .zig source):
```zig
const c = @cImport({
    @cInclude("stdio.h");
});
```

New API (in build.zig):
```zig
const c_module = b.addTranslateC(.{ .root_source_file = b.path("c_defs.h") });
exe.root_module.addImport("c", c_module);
```

Then in Zig source:
```zig
const c = @import("c");
```

### `switch` — Labeled Switch Improvements

Labeled `switch` with `continue` is now fully stabilized. Use it for state-machine style control flow.

```zig
state_machine: switch (state) {
    .start => {
        state = .running;
        continue :state_machine state;
    },
    .running => { ... },
}
```

### Vectors and Arrays No Longer Support In-Memory Coercion

You can no longer `@bitCast` or coerce between vectors/arrays of the same total size. Use explicit element-by-element copying or `@Vector`/ `@Array` builtins to construct types correctly.

### Forbid Runtime Vector Indexes

Vector indexing (`v[i]`) must now be comptime-known. Use `@shuffle`, `@select`, or scalar loops for dynamic vector element access.

### Forbid Trivial Local Address Returned from Functions

Returning the address of a local variable is now a compile error (previously it was allowed but UB). Ensure all returned pointers point to heap, global, or parameter memory.

### `@floor`, `@ceil`, `@round`, `@trunc` Convert to Integers

These builtins now return integer types when the result type is an integer, eliminating the need for a separate `@intFromFloat` cast.

```zig
const i: i32 = @floor(f); // returns i32 directly
```

### Packed Type Restrictions

- No pointers in packed structs/unions — remove any `*T` fields from packed types.
- Explicit backing integers required on packed unions — `packed union { ... }` is no longer valid without `packed union(u32) { ... }`.
- Unused bits forbidden in packed unions — all bits of the backing integer must be covered by fields.
- Enums and packed types with implicit backing types forbidden in extern contexts — use explicit backing integers for C interop.

### Lazy Field Analysis

Struct field types are now analyzed lazily. This speeds up compilation but means forward references may behave differently. Ensure field types are fully resolved by usage point.

### Pointers to Comptime-Only Types Are No Longer Comptime-Only

You can now have runtime pointers to comptime-only types (e.g., `*type`, `*comptime_int`). This unlocks new metaprogramming patterns.

### Explicitly-Aligned Pointer Types Now Distinct

`*align(8) T` and `*T` (where T is naturally 8-byte aligned) are now distinct types. You may need `@ptrCast` where you previously didn't.

## Build System

### `@cImport` Migration

As noted above, all C header translation moves to `build.zig`. Update `build.zig` files to use `b.addTranslateC()`.

### No Major New Build System Breaking Changes

0.16 focuses on stabilizing the 0.14/0.15 build system API. Most existing `build.zig` files from 0.15 should compile without changes, provided they already use explicit root modules.

## Agent Tips

- Always use interface-based I/O for new code. Functions should accept `std.Io` (or `std.Io.Writer`/`std.Io.Reader`) rather than `anytype` writers.
- Replace `@Type` with the specific type builtin (`@Int`, `@Struct`, etc.). This produces better error messages and clearer intent.
- Audit packed structs/unions for pointer fields and missing backing integers before upgrading.
- Vector indexing must be comptime-known; check any dynamic vector element access.
- Return local addresses will now error instead of silently producing UB. Good safety improvement, but may break code that accidentally relied on it.
- Migrate `@cImport` into `build.zig` during any upgrade; do not leave `@cImport` calls in new code.
