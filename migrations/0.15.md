# Migrating to Zig 0.15

Zig 0.15 (2025) is a high-impact release with breaking changes across the standard library, language, and build system. This guide covers the changes most likely to affect existing codebases.

## Standard Library

### I/O Overhaul ("Writergate")

The entire `std.io` abstraction layer was redesigned. Buffers moved into the interface, not the implementation.
Old API (pre-0.15):
```zig
const std = @import("std");
var gpa = std.heap.GeneralPurposeAllocator(.{}){};
const allocator = gpa.allocator();

var stdout = std.io.getStdOut().writer();
try stdout.print("Hello, {s}\n", .{"world"});
```

New API (0.15+):
```zig
const std = @import("std");

var stdout = std.fs.stdout();
var writer = stdout.writer();
try writer.print("Hello, {s}\n", .{"world"});
try stdout.flush(); // must flush explicitly
```

Key changes:
- `std.io.getStdOut()` / `std.io.getStdErr()` moved to `std.fs.stdout()` / `std.fs.stderr()`
- `std.io.getStdIn()` moved to `std.fs.stdin()`
- Writers/readers now require explicit `flush()` before program exit
- `CountingWriter` and `BufferedWriter` were deleted; use the new `std.Io.Writer`/`std.Io.Reader` interfaces instead

### ArrayList is Unmanaged by Default

`std.ArrayList` no longer stores an allocator internally.
Old API (pre-0.15):
```zig
var list = std.ArrayList(u8).init(allocator);
try list.append('a');
list.deinit();
```

New API (0.15+):
```zig
var list: std.ArrayListUnmanaged(u8) = .{};
try list.append(allocator, 'a');
list.deinit(allocator);
```

Storing in a struct:
```zig
const MyStruct = struct {
    list: std.ArrayListUnmanaged(u8) = .{},

    fn append(self: *MyStruct, allocator: std.mem.Allocator, item: u8) !void {
        try self.list.append(allocator, item);
    }
};
```

### Format String Changes

- `format` methods must be called with `"{f}"` specifier
- Format methods no longer accept format strings or options directly
- Formatted printing no longer handles Unicode; use `std.unicode` directly

Old API:
```zig
try std.fmt.format(writer, "{}", .{myStruct});
```

New API:
```zig
try std.fmt.format(writer, "{f}", .{myStruct});
```

### BoundedArray Removed

Use `std.BoundedArray` → replace with `std.ArrayListUnmanaged` with a fixed capacity, or manage buffers manually.

### Linked Lists De-Genericified

Linked list types (`std.SinglyLinkedList`, `std.DoublyLinkedList`) no longer require a type parameter. They operate on `*Node` directly.

### std.builtin.Type Fields Renamed (Lowercase)

All `std.builtin.Type` union tags are now lowercase. Use `@""` syntax for keyword conflicts.
Old API:
```zig
const info = @typeInfo(T);
if (info == .Int) { ... }
```

New API:
```zig
const info = @typeInfo(T);
if (info == .int) { ... }
if (info == .@"enum") { ... }
```

### HTTP Client and Server Reworked

The HTTP stack was overhauled. Use the new `std.http` APIs which now operate on the redesigned I/O interfaces.

### TLS Client

A TLS client was added to the standard library. If you previously relied on third-party TLS, evaluate migrating to `std.crypto.tls`.

## Language Changes

### `usingnamespace` Removed

The `usingnamespace` keyword was deleted. Replace with explicit imports or conditional compilation via `if` blocks at comptime.
Old API:
```zig
usingnamespace @import("common.zig");
```

New API:
```zig
const common = @import("common.zig");
```

### `async` and `await` Keywords Removed

The `async` and `await` keywords were removed from the language (planned resurrection in a future release with a different model). Remove any `async`/`await` usage; use callbacks or state machines in the interim.

### New Rules for Arithmetic on `undefined`

Only operators that can never trigger Illegal Behavior are allowed to take `undefined` as an operand. Review any arithmetic that previously relied on lazy `undefined` evaluation.

### Error on Lossy Coercion from Int to Float

Implicit lossy coercion from integer to float is now a compile error. Use an explicit cast:

```zig
const f: f32 = @floatFromInt(my_int);
```

### `@ptrCast` Allows Single-Item Pointer to Slice

You may now cast `*T` to `[]T` directly with `@ptrCast`, simplifying some pointer-to-slice transitions.

### `switch` on Non-Exhaustive Enums

Non-exhaustive enums now require an `else` prong in `switch` statements.

## Build System

### Removed Deprecated Implicit Root Module

Build scripts must now explicitly declare the root module. Use `b.createModule(.{ .root_source_file = b.path("src/main.zig") })` instead of implicit root file paths.

### `addStaticLibrary` Removed

Old API:
```zig
const lib = b.addStaticLibrary(.{ .name = "mylib", .root_source_file = b.path("src/lib.zig") });
```

New API:
```zig
const lib = b.addLibrary(.{ .name = "mylib", .linkage = .static, .root_module = module });
```

## Compiler

- x86 Backend: Selected by default for debug builds; ~5x faster debug compilation.
- aarch64 Backend: Work-in-progress, not default yet.
- Incremental Compilation: Available for faster edit/compile/debug cycles.
- UBSan Mode: Can be configured at the module level.

## Agent Tips

- When generating Zig code, always assume 0.15+ APIs for new projects.
- Explicit `flush()` on stdout/stderr is a common omission; warn users to include it before program exit.
- ArrayList operations now need allocator parameters everywhere; double-check struct signatures.
- `"{f}"` is the correct specifier for custom `format` methods—`"{}"` will not invoke them.
